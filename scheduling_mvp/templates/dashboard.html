🔬 Entrenamiento del MLP (Fase 4)

¿Qué es el entrenamiento? Es el proceso donde la red neuronal aprende de los datos. En cada época (epoch), la red procesa todos los datos de entrenamiento, calcula el error y ajusta sus pesos internos para mejorar las predicciones. Después de varias épocas, la red debería cometer menos errores.

📉 Loss (Pérdida) por Época

Loss (Binary Crossentropy): Mide el error entre la predicción de la red y la respuesta correcta. Train Loss = error en datos de entrenamiento. Val Loss = error en datos que la red NO vio durante el entrenamiento (validación). Ambas curvas deben bajar. Si Val Loss sube mientras Train baja → sobreajuste (overfitting).

📈 Accuracy (Precisión) por Época

Accuracy: Porcentaje de veces que la red predijo correctamente si un empleado debe o no ser asignado a un turno. Train Acc = precisión en datos de entrenamiento. Val Acc = precisión en datos de validación. Un valor cercano a 1.0 (100%) es ideal.

🧬 Factores Importantes en una MLP

Una red MLP tiene varios hiperparámetros que afectan su rendimiento. A continuación se detalla cada factor con su configuración en este proyecto y su justificación técnica.

🔧 Hiperparámetros de Arquitectura

Número de Capas Ocultas

Valor usado: 3 capas — Suficiente para capturar relaciones no lineales entre disponibilidad, horas y turnos sin sobreajustar con tan pocos datos.

Neuronas por Capa

64 → 32 → 16 (embudo) — Patrón descendente que comprime la información progresivamente, forzando a la red a extraer las features más relevantes.

Tasa de Aprendizaje (Learning Rate)

0.001 (default Adam) — Adam adapta automáticamente la tasa por cada parámetro, lo que lo hace robusto sin necesidad de ajuste manual.

Cantidad de Datos

20,000 muestras (500 soluciones × 5 empleados × 8 turnos) — Generadas sintéticamente con variación estocástica de la heurística para dar diversidad al entrenamiento.

⚡ Funciones de Activación

Las funciones de activación introducen no linealidad en la red. Sin ellas, la red sería equivalente a un modelo lineal simple, incapaz de resolver problemas complejos como el scheduling.

✅ ReLU — USADA en capas ocultas

Rectified Linear Unit: f(x) = max(0, x). Es la función más usada en deep learning por su eficiencia computacional y porque mitiga el desvanecimiento del gradiente.

✅ Sigmoid — USADA en capa de salida

Sigmoide: f(x) = 1/(1+e⁻ˣ). Salida entre 0 y 1 → se interpreta como probabilidad de asignación. Perfecta para clasificación binaria (¿asignar o no?).

❌ Tanh — No usada

Tangente Hiperbólica: f(x) = tanh(x). Rango [-1, 1]. Más centrada que sigmoid pero susceptible al desvanecimiento del gradiente. No la usamos porque ReLU es más eficiente para este caso.

¿Por qué esta combinación? ReLU en capas ocultas porque es rápida, no sufre desvanecimiento del gradiente y funciona excelente con Adam. Sigmoid en la salida porque necesitamos una probabilidad entre 0 y 1 para decidir si asignar o no un empleado a un turno.

⚠️ Problemas Comunes en MLP y Cómo se Mitigan

Sobreajuste (Overfitting)
Mitigado con validación 80/20 y pocas épocas. Val Loss ~= Train Loss ✓
Subajuste (Underfitting)
No presente: 3 capas con 64-32-16 neuronas dan suficiente capacidad. Accuracy >93% ✓
Desvanecimiento del Gradiente
Mitigado usando ReLU en capas ocultas (gradiente constante para x>0) ✓
