Análisis Lógico: modelo_mlp.py

Desglose arquitectónico y funcional de la Red Neuronal para Scheduling

entrenar_mlp()

Construcción y optimización del modelo

Esta función es el núcleo de aprendizaje. Su objetivo es tomar un conjunto de datos (features y labels) y ajustar los pesos de la red neuronal para que aprenda a relacionar el perfil de un empleado con la idoneidad para un turno.

Lógica de Ejecución

  • Definición Arquitectónica: Instancia un modelo `Sequential` de Keras. Crea una estructura de embudo (64 → 32 → 16 neuronas) con activación `ReLU` para capturar relaciones no lineales complejas entre las características del turno y el empleado.
  • Capa de Salida: Utiliza una única neurona con activación `Sigmoid` para garantizar que la salida sea estrictamente una probabilidad (valor entre 0.0 y 1.0).
  • Compilación: Asigna `adam` como optimizador (eficiente para ajuste de pesos) y `binary_crossentropy` como función de pérdida, la cual es estándar para clasificación binaria (asignar o no asignar).
  • Validación Cruzada Simple: Divide el dataset estáticamente en 80% entrenamiento y 20% validación para monitorear el sobreajuste (overfitting).
  • Entrenamiento: Ejecuta el método `.fit()`, iterando `epochs` veces sobre los datos en lotes (`batch_size`), y evalúa métricas finales.

predecir_asignacion()

Inferencia y aplicación de reglas de negocio operativas

El modelo matemático por sí solo no garantiza una asignación legal (puede predecir que alguien trabaje 100 horas si las features son engañosas). Esta función actúa como el puente entre la IA probabilística y la realidad operativa mediante una estrategia *Greedy* post-predicción.

Lógica de Ejecución

  • Generación de Matriz de Posibilidades: Crea el producto cartesiano de todos los empleados y todos los turnos. Extrae el vector de features para cada par exacto.
  • Inferencia Batch: Pasa toda la matriz (`X_pred`) por el MLP entrenado en un solo lote para obtener las probabilidades de éxito de cada par.
  • Ordenamiento de Prioridad: Ordena la lista completa de pares de mayor a menor probabilidad. Esto asegura que el algoritmo priorice las combinaciones que la IA considera más "ideales".
  • Filtro Hard-Constraints: Itera sobre la lista ordenada. Antes de asignar, pasa por validadores estrictos: ¿El turno está lleno? ¿El empleado está disponible? ¿Supera sus horas máximas? ¿Ya trabaja ese día? Si falla alguna, se descarta la asignación sin importar qué tan alta fuera la probabilidad de la red neuronal.
  • Registro y Exportación: Construye el estado final de las asignaciones, calcula las horas consumidas y exporta la estructura a `asignacion_mlp.json`.

guardar_historial()

Persistencia de métricas de telemetría

Extrae las métricas del objeto `History` devuelto por Keras (loss, accuracy, val_loss, val_accuracy) al final del entrenamiento y las serializa en un formato estándar JSON. Su utilidad radica en permitir la posterior graficación de las curvas de aprendizaje sin necesidad de retener el modelo en memoria RAM.
